# Σύστημα Παρακολούθησης Παραγγελιών - OrderTrack

## 1. Σύντομη Περιγραφή της Εφαρμογής

Το OrderTrack είναι μια διαδικτυακή εφαρμογή που αναπτύχθηκε για να καλύψει το κενό ενημέρωσης μεταξύ των επιχειρήσεων και των πελατών τους σχετικά με την εξέλιξη των παραγγελιών. Το βασικό πρόβλημα που επιλύει είναι η έλλειψη διαφάνειας μετά την αγορά, επιτρέποντας στους χρήστες να γνωρίζουν ανά πάσα στιγμή σε ποιο στάδιο βρίσκεται η παραγγελία τους χωρίς να χρειάζεται να επικοινωνήσουν τηλεφωνικά με το κατάστημα.

Οι κύριοι χρήστες της εφαρμογής είναι οι Πελάτες, οι οποίοι μπορούν να αναζητήσουν την πορεία της αποστολής τους και να δουν το ιστορικό των αγορών τους, και οι Διαχειριστές, οι οποίοι είναι υπεύθυνοι για την ενημέρωση της κατάστασης των παραγγελιών. 

Σημείωση για τη Βασική Ιδέα (Συγκριτικά με το 1ο Παραδοτέο):
Δεν υπήρξε καμία αλλαγή στην κεντρική ιδέα της εφαρμογής. Ωστόσο, στην πράξη οι λειτουργίες της επεκτάθηκαν και θωρακίστηκαν με την προσθήκη αυστηρού ελέγχου δικαιωμάτων (Authorization - 403 Forbidden) για την προστασία των δεδομένων, και εμπλουτίστηκαν με διαδραστικά στοιχεία διεπαφής (Custom JavaScript / Confetti Effect) για καλύτερη εμπειρία χρήστη.

Κύριοι Χρήστες:
* Πελάτες: Αναζητούν την πορεία της αποστολής τους και καταχωρούν νέες παραγγελίες.
* Διαχειριστές: Ενημερώνουν την κατάσταση των παραγγελιών μέσω του ασφαλούς Django Admin Panel.

## 2. Πώς Εκτελείται η Εφαρμογή

Για να τρέξετε την εφαρμογή τοπικά στον υπολογιστή σας, ακολουθήστε τα παρακάτω βήματα στο τερματικό:

1. Ενεργοποίηση εικονικού περιβάλλοντος (Virtual Environment):
   Για Windows: venv\Scripts\activate
   Για Mac/Linux: source venv/bin/activate

2. Εγκατάσταση εξαρτήσεων (Django):
   pip install django

3. Εκτέλεση των Migrations (Βάση Δεδομένων SQLite):
   python manage.py makemigrations
   python manage.py migrate

4. Εκκίνηση του Server:
   python manage.py runserver

Η εφαρμογή θα είναι διαθέσιμη στη διεύθυνση: http://127.0.0.1:8000/

## 3. Περιγραφή και Τεκμηρίωση Λειτουργιών (Οδηγίες Χρήσης)

### Λειτουργία 1: Εγγραφή Χρήστη (Sign-up)
* Περιγραφή: Επιτρέπει σε νέους πελάτες να δημιουργήσουν λογαριασμό.
* Οδηγίες Εκτέλεσης: Πλοηγηθείτε στο /signup/. Συμπληρώστε τα στοιχεία σας (Όνομα χρήστη, Email, Κωδικό και Επιβεβαίωση) και πατήστε το κουμπί «Ολοκλήρωση Εγγραφής».
* Αναμενόμενο Αποτέλεσμα: Δημιουργείται ο χρήστης στη βάση δεδομένων και εμφανίζεται μήνυμα επιτυχίας.
* HTTP Method/URL: POST /signup/ | Template: templates/signup.html
* [Σελίδα Σύνδεσης](screenshots/signup.png)

### Λειτουργία 2: Σύνδεση Χρήστη (Login)
* Περιγραφή: Οι εγγεγραμμένοι χρήστες αποκτούν πρόσβαση στο προσωποποιημένο περιβάλλον τους.
* Οδηγίες Εκτέλεσης: Πλοηγηθείτε στο /login/. Γράψτε το Όνομα Χρήστη και τον Κωδικό σας στα αντίστοιχα πεδία. Πατήστε το κουμπί «Είσοδος στο Σύστημα».
* Αναμενόμενο Αποτέλεσμα: Η εφαρμογή επικυρώνει τα στοιχεία (χειρισμός λαθών με error_message αν είναι λανθασμένα) και σας ανακατευθύνει στο Ιστορικό Παραγγελιών.
* HTTP Method/URL: POST /login/ | Template: templates/login.html
* [Σελίδα Σύνδεσης](screenshots/login.png)

### Λειτουργία 3: Διαχείριση Προφίλ
* Περιγραφή: Σελίδα όπου οι χρήστες βλέπουν τα προσωπικά τους στοιχεία.
* HTTP Method/URL: GET /profile/ | Template: templates/profile.html
* [Σελίδα Σύνδεσης](screenshots/profile.png)

### Λειτουργία 4: Δημιουργία Παραγγελίας
* Περιγραφή: Καταχώρηση μιας νέας παραγγελίας στο σύστημα.
* Οδηγίες Εκτέλεσης: Από το μενού πατήστε «Νέα Παραγγελία» (/orders/new/). Συμπληρώστε τα προϊόντα, τις ποσότητες και τις τιμές. Πατήστε «Καταχώρηση Παραγγελίας».
* Αναμενόμενο Αποτέλεσμα: Το σύστημα ελέγχει τα δεδομένα, υπολογίζει αυτόματα το συνολικό κόστος, παράγει ένα μοναδικό Tracking ID και αποθηκεύει την παραγγελία.
* HTTP Method/URL: POST /orders/new/ | Template: templates/order_form.html
* [Σελίδα Σύνδεσης](screenshots/new.png)

### Λειτουργία 5: Προβολή Λίστας Παραγγελιών
* Περιγραφή: Εμφάνιση πίνακα με όλες τις παραγγελίες που ανήκουν αποκλειστικά στον συνδεδεμένο χρήστη.
* HTTP Method/URL: GET /orders/ | Template: templates/order_list.html
* [Σελίδα Σύνδεσης](screenshots/orders.png)

### Λειτουργία 6: Λεπτομέρειες Παραγγελίας 
* Περιγραφή: Αναλυτική προβολή των προϊόντων και της πορείας μιας συγκεκριμένης παραγγελίας με χρήση Bootstrap Progress Bar.
* Οδηγίες Εκτέλεσης: Στον πίνακα της λίστας παραγγελιών (`/orders/`), εντοπίστε την παραγγελία που θέλετε και πατήστε το κουμπί Προβολή. 
* Αναμενόμενο Αποτέλεσμα: Το σύστημα ανακτά τα προϊόντα της συγκεκριμένης παραγγελίας και εμφανίζει μια δυναμική μπάρα προόδου (Progress Bar) που δείχνει αν η παραγγελία είναι "Σε επεξεργασία", "Απεστάλη" ή "Παραδόθηκε". (Αν προσπαθήσετε να μπείτε σε ID παραγγελίας άλλου χρήστη, το σύστημα πετάει σφάλμα 403 Forbidden)
* HTTP Method/URL: GET /orders/<int:order_id>/ | Template: templates/order_detail.html
* [Σελίδα Σύνδεσης](screenshots/details.png)

### Λειτουργία 7: Αναζήτηση βάσει Tracking ID (Custom JavaScript & Confetti)
* Περιγραφή: Δημόσια αναζήτηση της κατάστασης μιας παραγγελίας χωρίς ανάγκη σύνδεσης.
* Οδηγίες Εκτέλεσης: Πλοηγηθείτε στη σελίδα εντοπισμού (`/track/`). Πληκτρολογήστε ένα έγκυρο Tracking ID στο πεδίο αναζήτησης και πατήστε το κουμπί «Αναζήτηση».
* Αναμενόμενο Αποτέλεσμα: 1. Μόλις πατηθεί το κουμπί, ενεργοποιείται JavaScript που απενεργοποιεί το κουμπί και εμφανίζει ένα εικονίδιο φόρτωσης (loading spinner) για να αποφευχθούν διπλές υποβολές.
  2. Αν η παραγγελία βρεθεί επιτυχώς, εμφανίζονται τα στοιχεία της και ενεργοποιείται δεύτερη JavaScript που εκτοξεύει πολύχρωμα κονφετί στην οθόνη! Αν δεν βρεθεί, το σύστημα εμφανίζει ελεγχόμενο μήνυμα σφάλματος.
* HTTP Method/URL: GET & POST /track/ | Template: `templates/track.html`
* [Σελίδα Σύνδεσης](screenshots/track.png)

## 4. Σχήμα Βάσης Δεδομένων

Το σχήμα της βάσης δεδομένων βασίζεται στο σχεσιακό μοντέλο και υλοποιείται μέσω του Django ORM:

1. UserProfile (1:1 με το User του Django): Αποθηκεύει επιπλέον στοιχεία (τηλέφωνο, διεύθυνση).
2. Order (1:Ν με το User): Κάθε παραγγελία ανήκει σε έναν χρήστη, έχει ένα μοναδικό tracking_id, ημερομηνία created_at και status (Processing, Shipped, Delivered).
3. OrderItem (1:Ν με το Order): Σπάει τη σχέση Many-to-Many, αποθηκεύοντας product_name, quantity και price για κάθε προϊόν μιας παραγγελίας.

## 5. Περιορισμοί της Εφαρμογής & Παρατηρήσεις

* Η Ενημέρωση Κατάστασης (Update) και η Ακύρωση/Διαγραφή (Delete) των παραγγελιών γίνονται προς το παρόν αποκλειστικά από την πλευρά του διαχειριστή μέσω του έτοιμου και ασφαλούς Django Admin Panel (/admin/), καθώς αυτό προσομοιώνει το πραγματικό backend σύστημα μιας εταιρείας courier. Τα mock-up templates order_status_update.html και order_cancel.html δεν χρησιμοποιούνται στο frontend για λόγους ασφαλείας.
* Ασφάλεια Δεδομένων (403 Forbidden): Έχει υλοποιηθεί έλεγχος στις Views, ώστε αν ένας συνδεδεμένος χρήστης αλλάξει χειροκίνητα το ID στο URL για να δει παραγγελία άλλου χρήστη, η εφαρμογή να το μπλοκάρει αμέσως επιστρέφοντας σφάλμα ασφαλείας.
* Οι αρχικές λειτουργίες φιλτραρίσματος και ταξινόμησης ενοποιήθηκαν για λόγους βέλτιστης αρχιτεκτονικής μέσα στην κεντρική προβολή των παραγγελιών.
